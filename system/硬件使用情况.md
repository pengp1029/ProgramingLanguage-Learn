# 如何利用系统指令分析当前机器的负载情况
## cpu、内存负载
`top -c`：进程级别cpu、memory使用情况
1. 系统概览
- load average：1min、5min、15min平均负载
- Tasks：进程统计
- %Cpu(s)：CPU使用率（us用户态、sy内核态、ni用户态、id空闲、wa等待IO设备操作、hi硬件中断、si软件中断、st偷走）
- MiB Mem：memory使用情况（total总物理内存、free完全空闲内存、used已使用内存、shrd共享内存、buff/cache缓冲和缓存内存）
2. 进程列表
- PID：进程ID
- USER：进程所有者
- PR：优先级（普通进程为20，数值越大优先级越低）
- NI：nice值（调整优先级的相对值）
- VIRT：进程使用的虚拟内存总量，包括物理内存（RES）
- RES：进程当前使用的物理内存大小
- SHR：进程使用的共享内存大小，包括与其他进程共享的库文件、共享内存段等
- S：进程状态（R运行、S睡眠、T停止、Z僵尸）
- %CPU：进程使用的CPU百分比，指的是进程占用CPU时间片的比例
- %MEM：进程占用的物理内存占系统总内存比例
- TIME+：进程占用的CPU时间总计
- COMMAND：启动进程的命令
